Piece_Unity Examples

A. Continuation Server in Single Flow Mode

1. Registration application in single flow mode.

2. A static page in single flow mode. This approach requires only one template in the template directory.

3. A static page with a simple action in single flow mode. This approach requires a template in the template directory and an action in the action directory.

B. Continuation Server in Multiple Flow Mode

1. Registration application in multiple flow mode and flow in non-exclusive mode.

2. Counter application in multiple flow mode and flow in exclusive mode.

3. A static page in multiple flow mode. This approach requires only one template in the template directory.

4. A static page with a simple action in multiple flow mode. This approach requires a template in the template directory and an action in the action directory.
